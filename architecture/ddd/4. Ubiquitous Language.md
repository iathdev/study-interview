# Ubiquitous Language

Ubiquitous Language (Ngôn ngữ phổ quát) là **ngôn ngữ chung được dùng thống nhất** giữa domain expert và developer trong cùng một Bounded Context — trong cuộc họp, trong tài liệu, và trong code.

> "Ubiquitous" nghĩa là "có mặt khắp nơi" — không phải developer dùng một ngôn ngữ, business dùng một ngôn ngữ, rồi có người phiên dịch ở giữa. Chỉ có **một ngôn ngữ duy nhất**.

---

## 1. Vấn đề khi không có Ubiquitous Language

```
Domain Expert nói:   "Khách hàng đặt chỗ, giữ chỗ 15 phút, sau đó xác nhận."
Developer nghĩ:      "User tạo booking với status PENDING, sau 15 phút tự CONFIRM."
Code viết:           userService.createBooking() → booking.setStatus("1")

Kết quả:
- Họp hành tốn thời gian dịch qua dịch lại
- Bug vì hiểu sai requirement
- Code không ai đọc được nếu không biết "status 1 là gì"
```

---

## 2. Xây dựng Ubiquitous Language

### Bước 1: Lắng nghe domain expert
Chú ý từ ngữ họ dùng tự nhiên. Nếu họ nói "hold" chứ không phải "reserve" → dùng "hold" trong code.

### Bước 2: Làm rõ thuật ngữ mơ hồ
```
Developer: "Khi nào thì booking được confirm?"
Expert:    "Khi thanh toán thành công."
Developer: "Nếu thanh toán fail thì booking đó đi đâu?"
Expert:    "Nó vẫn ở trạng thái held cho đến khi hết hạn 15 phút."
```

Từ cuộc hội thoại này → Ubiquitous Language có: `hold`, `confirm`, `expire`, không có `pending`, `active`.

### Bước 3: Đưa ngôn ngữ vào code ngay lập tức

```java
// Trước — không dùng Ubiquitous Language
booking.setStatus(1);           // 1 là gì?
if (booking.getStatus() == 2)   // 2 là gì?

// Sau — dùng đúng ngôn ngữ domain
reservation.hold(Duration.ofMinutes(15));
if (reservation.isConfirmed())
```

## 3. Ví dụ — Reservation BC trong đặt vé máy bay

### Từ điển thuật ngữ

| Thuật ngữ | Ý nghĩa trong Reservation BC | KHÔNG dùng |
|-----------|------------------------------|------------|
| **Hold** | Giữ chỗ tạm thời, có thời hạn | `pending`, `lock`, `reserve` |
| **Confirm** | Xác nhận sau thanh toán thành công | `approve`, `activate`, `finalize` |
| **Expire** | Hết hạn hold mà chưa confirm | `timeout`, `cancel automatically` |
| **Cancel** | Chủ động hủy bởi khách hàng | `delete`, `remove` |
| **Passenger** | Người thực sự đi trên máy bay | `user`, `customer`, `traveler` |
| **Contact** | Người đặt vé (có thể đặt cho người khác) | `booker`, `user` |
| **Inventory** | Số ghế còn lại theo hạng | `capacity`, `available_count` |
| **Fare class** | Hạng vé chi tiết (Y, B, M...) | `ticket_type`, `class` |

### Ubiquitous Language trong code

```java
// Class, method, variable đều dùng đúng ngôn ngữ domain
public class Reservation {
    private ReservationStatus status; // HELD, CONFIRMED, EXPIRED, CANCELLED

    public void hold(Duration duration) { ... }
    public void confirm(Payment payment) { ... }
    public void cancel(CancellationReason reason) { ... }
    public boolean isExpired() { ... }
    public boolean isHeld() { ... }
}

// Enum dùng đúng từ
public enum ReservationStatus {
    HELD, CONFIRMED, EXPIRED, CANCELLED
    // KHÔNG phải: PENDING, ACTIVE, DELETED, INACTIVE
}

// Event dùng đúng từ
public class ReservationHeld { ... }     // KHÔNG: BookingCreated
public class ReservationConfirmed { ... } // KHÔNG: BookingApproved
public class ReservationExpired { ... }   // KHÔNG: BookingTimeout
```

## 4. Ubiquitous Language khác nhau giữa các Bounded Context

Cùng khái niệm, tên khác nhau ở các BC khác nhau — và điều này **đúng**:

| Concept | Reservation BC | Customer Support BC | Loyalty BC |
|---------|---------------|---------------------|------------|
| Người dùng | `Passenger` / `Contact` | `Complainant` | `Member` |
| Chuyến bay | `Flight` (có inventory) | `Flight` (trong complaint) | `EarningFlight` |
| Trạng thái | `HELD`, `CONFIRMED` | `OPEN`, `RESOLVED` | `PENDING_CREDIT` |

Mỗi BC có **từ điển riêng**. Không cần và không nên đồng nhất tên giữa các BC.

## 5. Dấu hiệu Ubiquitous Language đang bị vi phạm

```java
// Tên kỹ thuật, không phải tên domain
BookingManager, DataProcessor, StatusUpdater

// Tên chung chung
handleRequest(), processData(), updateRecord()

// Comment giải thích code thay vì code tự giải thích
// status = 1 means HELD
// status = 2 means CONFIRMED

// Developer và expert dùng từ khác nhau trong cùng cuộc họp
Developer: "Khi booking confirmed..."
Expert:    "Ý bạn là khi reservation được xác nhận?"
```

## 6. Duy trì Ubiquitous Language

- **Glossary**: Duy trì file từ điển thuật ngữ cho từng BC (có thể trong CLAUDE.md)
- **Code review**: Reject PR nếu dùng sai thuật ngữ

- **Tài liệu**: Dùng đúng từ trong ticket, comment, PR description
