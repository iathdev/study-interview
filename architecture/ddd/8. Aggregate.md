# Aggregate

Aggregate là **cụm các Entity và Value Object** được nhóm lại thành một đơn vị nhất quán. Aggregate có một **Aggregate Root** — entry point duy nhất để tương tác với toàn bộ cụm.

> Aggregate là ranh giới consistency: mọi invariant (quy tắc bất biến) bên trong Aggregate phải luôn được đảm bảo sau mỗi transaction.

---

## 1. Đặc điểm

| Đặc điểm | Mô tả |
|----------|-------|
| **Aggregate Root** | Entity trung tâm, là điểm vào duy nhất |
| **Consistency boundary** | Một transaction chỉ thay đổi một Aggregate |
| **Identity từ Root** | Aggregate được xác định bằng ID của Root |
| **Encapsulation** | Bên ngoài không truy cập trực tiếp vào các object con |
| **Reference nội bộ bằng object** | Root giữ child Entity bằng object reference (`List<ReservationItem>`) |
| **Reference bên ngoài bằng ID** | Aggregate này reference Aggregate khác qua ID, không qua object |
| **Raise Domain Events** | Aggregate Root collect và publish Domain Event khi có state transition |

---

## 2. Ví dụ — Reservation Aggregate

```
Reservation (Aggregate Root)
├── List<ReservationItem>     (Entity — mỗi hành khách/ghế)
├── Money totalPrice          (Value Object)
├── ReservationStatus status  (Value Object / Enum)
└── ContactInfo contact       (Value Object)
```

```java
public class Reservation {  // Aggregate Root
    private final ReservationId id;
    private final List<ReservationItem> items;  // Entity bên trong
    private Money totalPrice;
    private ReservationStatus status;
    private ContactInfo contact;

    // Bên ngoài KHÔNG thể new ReservationItem trực tiếp
    // Phải đi qua Aggregate Root
    public void addPassenger(PassengerId passengerId, SeatId seatId, Money price) {
        validateNotConfirmed();
        validateSeatNotDuplicated(seatId);

        ReservationItem item = new ReservationItem(passengerId, seatId, price);
        items.add(item);
        this.totalPrice = totalPrice.add(price);  // Root giữ totalPrice nhất quán
    }

    public void removePassenger(PassengerId passengerId) {
        ReservationItem item = findItemByPassenger(passengerId);
        items.remove(item);
        this.totalPrice = totalPrice.subtract(item.getPrice());
    }

    public void confirm(Payment payment) {
        if (items.isEmpty()) {
            throw new DomainException("Không thể confirm đặt chỗ rỗng");
        }
        if (!payment.getAmount().equals(totalPrice)) {
            throw new DomainException("Số tiền thanh toán không khớp");
        }
        this.status = ReservationStatus.CONFIRMED;
    }

    // Invariant: đảm bảo không có ghế trùng
    private void validateSeatNotDuplicated(SeatId seatId) {
        boolean alreadyBooked = items.stream()
            .anyMatch(item -> item.getSeatId().equals(seatId));
        if (alreadyBooked) {
            throw new DomainException("Ghế đã được đặt trong reservation này");
        }
    }
}
```

## 3. Quy tắc thiết kế Aggregate

### Rule 1: Bên trong Aggregate — giữ object. Bên ngoài — chỉ giữ ID

Root giữ child Entity bằng **object reference** vì chúng cùng sống trong một Aggregate, cùng transaction, cùng consistency boundary:

```java
public class Reservation {
    private List<ReservationItem> items; // object reference — đúng, vì items là child entity của Reservation
}
```

Nhưng khi reference sang **Aggregate khác** — chỉ giữ ID:

### Rule 1: Chỉ reference Aggregate khác bằng ID

```java
// Tệ — giữ object reference → tạo coupling chặt, transaction phức tạp
public class Reservation {
    private Flight flight;           // Object reference → sai
    private Passenger passenger;     // Object reference → sai
}

// Tốt — chỉ giữ ID
public class Reservation {
    private FlightId flightId;       // ID reference → đúng
    private PassengerId passengerId; // ID reference → đúng
}
```

### Rule 2: Một transaction, một Aggregate

```java
// Tệ — một transaction thay đổi 2 Aggregate
@Transactional
public void confirmReservation(ReservationId id) {
    Reservation reservation = reservationRepo.findById(id);
    reservation.confirm();

    Flight flight = flightRepo.findById(reservation.getFlightId());
    flight.decreaseSeatCount();  // ← thay đổi Aggregate khác trong cùng transaction
}

// Tốt — dùng Domain Event để xử lý bất đồng bộ
@Transactional
public void confirmReservation(ReservationId id) {
    Reservation reservation = reservationRepo.findById(id);
    reservation.confirm();  // publish event: ReservationConfirmed
    reservationRepo.save(reservation);
    // Flight BC sẽ nhận event và tự cập nhật số ghế
}
```

### Rule 3: Aggregate nên nhỏ

Aggregate lớn → lock nhiều → performance kém → deadlock.

```
// Tệ — Order Aggregate quá lớn
Order (Root)
├── List<OrderItem>
├── Customer        ← Entity riêng → nên tham chiếu bằng CustomerId
├── Payment         ← Aggregate riêng
└── ShippingAddress

// Tốt — tách nhỏ, reference bằng ID
Order (Root)         Reservation Aggregate
├── List<OrderItem>  Customer Aggregate (CustomerId)
├── CustomerId ──────────────────────────────────▶
└── ShippingAddress
```

## 4. Aggregate trong hệ thống đặt vé máy bay

### Reservation Aggregate
```
Reservation (Root)
├── ReservationId
├── FlightId            (ref đến Flight Aggregate)
├── List<ReservationItem>
│   ├── PassengerId     (ref đến Passenger Aggregate)
│   ├── SeatId
│   └── Money price
├── Money totalPrice
├── ReservationStatus
└── ContactInfo
```

### Flight Aggregate
```
Flight (Root)
├── FlightId
├── AirportCode origin
├── AirportCode destination
├── LocalDateTime departureTime
├── FlightStatus
└── SeatInventory
    ├── int totalSeats
    └── Map<SeatClass, Integer> availableByClass
```

### Passenger Aggregate
```
Passenger (Root)
├── PassengerId
├── PersonName
├── Email
├── PhoneNumber
└── FrequentFlyerNumber (nullable)
```

## 5. Khi nào nên tách thành Aggregate riêng?

Tách ra khi:
- Hai object có **vòng đời độc lập** (tạo, xóa riêng)
- Một object có thể tồn tại **không cần** object kia
- Hai object cần được **truy cập độc lập** từ nhiều nơi

Giữ chung khi:
- Không thể tồn tại nếu thiếu Root (ReservationItem không có nghĩa nếu không có Reservation)
- Invariant phải được đảm bảo nguyên tử (thêm item phải cập nhật totalPrice ngay)
