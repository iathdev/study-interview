# Value Object

Value Object là object được xác định hoàn toàn bởi **giá trị của các thuộc tính**, không có identity riêng. Hai Value Object bằng nhau khi tất cả thuộc tính bằng nhau.

> Nếu bạn thay tờ 100k bằng tờ 100k khác — bạn vẫn có 100k. Tiền là Value Object.
> Nếu bạn thay người dùng A bằng người dùng B — hoàn toàn khác. User là Entity.

---

## 1. Đặc điểm

| Đặc điểm | Mô tả |
|----------|-------|
| **Không có identity** | Không có ID |
| **Immutable** | Không thể thay đổi sau khi tạo — chỉ tạo mới |
| **Equality theo giá trị** | `vo1 == vo2` khi tất cả thuộc tính bằng nhau |
| **Self-validating** | Tự validate khi khởi tạo |
| **Có behavior** | Có thể có method tính toán, không được thay đổi state |

## 2. Ví dụ — Hệ thống đặt vé máy bay

### Money

```java
public final class Money {
    private final BigDecimal amount;
    private final Currency currency;

    public Money(BigDecimal amount, Currency currency) {
        if (amount.compareTo(BigDecimal.ZERO) < 0) {
            throw new IllegalArgumentException("Số tiền không được âm");
        }
        this.amount = amount;
        this.currency = currency;
    }

    public Money add(Money other) {
        if (!this.currency.equals(other.currency)) {
            throw new DomainException("Không thể cộng 2 loại tiền tệ khác nhau");
        }
        return new Money(this.amount.add(other.amount), this.currency); // trả về object mới
    }

    public Money multiply(int quantity) {
        return new Money(this.amount.multiply(BigDecimal.valueOf(quantity)), this.currency);
    }

    @Override
    public boolean equals(Object o) {
        if (!(o instanceof Money)) return false;
        Money other = (Money) o;
        return amount.equals(other.amount) && currency.equals(other.currency);
    }
}

// Dùng
Money price = new Money(new BigDecimal("500000"), Currency.getInstance("VND"));
Money tax = new Money(new BigDecimal("50000"), Currency.getInstance("VND"));
Money total = price.add(tax); // 550000 VND — object mới, price và tax không đổi
```

### SeatClass

```java
public enum SeatClass {
    ECONOMY("Y"), PREMIUM_ECONOMY("W"), BUSINESS("C"), FIRST("F");

    private final String code;

    SeatClass(String code) { this.code = code; }

    public boolean isUpgradeFrom(SeatClass other) {
        return this.ordinal() > other.ordinal();
    }
}
```

### DateRange (Khoảng thời gian)

```java
public final class DateRange {
    private final LocalDate from;
    private final LocalDate to;

    public DateRange(LocalDate from, LocalDate to) {
        if (to.isBefore(from)) {
            throw new IllegalArgumentException("Ngày kết thúc phải sau ngày bắt đầu");
        }
        this.from = from;
        this.to = to;
    }

    public boolean overlaps(DateRange other) {
        return !this.to.isBefore(other.from) && !other.to.isBefore(this.from);
    }

    public long durationInDays() {
        return ChronoUnit.DAYS.between(from, to);
    }
}
```

### AirportCode

```java
public final class AirportCode {
    private final String code;

    public AirportCode(String code) {
        if (code == null || !code.matches("[A-Z]{3}")) {
            throw new IllegalArgumentException("Mã sân bay phải là 3 ký tự viết hoa: " + code);
        }
        this.code = code;
    }

    public static final AirportCode HAN = new AirportCode("HAN");
    public static final AirportCode SGN = new AirportCode("SGN");
}
```

## 3. Khi nào dùng Value Object thay vì primitive

```java
// Tệ — primitive type, dễ nhầm lẫn thứ tự tham số
void bookFlight(String origin, String destination, int passengers) { }
bookFlight("SGN", "HAN", 2); // Đúng hay sai? Khó kiểm tra

// Tốt — Value Object tự validate, code tự documenting
void bookFlight(AirportCode origin, AirportCode destination, PassengerCount count) { }
bookFlight(AirportCode.SGN, AirportCode.HAN, new PassengerCount(2));
```

## 4. Value Object vs Entity

| | Value Object | Entity |
|--|-------------|--------|
| **Identity** | Không có | Có ID |
| **Equality** | Theo giá trị thuộc tính | Theo ID |
| **Mutability** | Immutable | Mutable |
| **Thay thế** | Có thể thay bằng object cùng giá trị | Không thể thay thế |
| **Ví dụ** | Money, Address, Email, DateRange | User, Order, Flight |

---

## 5. Lưu ý

- **Immutability là bắt buộc**: Không có setter, mọi "thay đổi" phải tạo object mới.
- **Value Object nên nhỏ**: Nếu quá phức tạp, có thể là Entity bị nhầm.
- **Dùng Value Object nhiều hơn Entity**: Code sẽ đơn giản hơn, ít bug về state hơn.
- **Lưu vào DB**: Thường được embed vào bảng của Entity (không có bảng riêng).
