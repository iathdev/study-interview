# Domain Service

Domain Service chứa **business logic không thuộc về bất kỳ Entity hay Value Object nào**. Khi một nghiệp vụ liên quan đến nhiều object hoặc không tự nhiên thuộc về một object cụ thể, đó là lúc cần Domain Service.

> Nếu không biết đặt logic vào Entity nào — đó thường là Domain Service.

---

## 1. Đặc điểm

| Đặc điểm | Mô tả |
|----------|-------|
| **Stateless** | Không giữ state, chỉ thực thi logic |
| **Tên theo domain** | Đặt tên theo hành động nghiệp vụ, không phải kỹ thuật |
| **Phụ thuộc vào domain** | Chỉ làm việc với Entity, Value Object, Repository |
| **Không phải Application Service** | Không xử lý HTTP, không gọi external API |

---

## 2. Dấu hiệu cần Domain Service

- Logic cần **nhiều Aggregate** cộng tác để ra quyết định
- Logic không "tự nhiên" thuộc về một Entity cụ thể
- Logic là **policy/rule** của domain, không phải behavior của object

---

## 3. Ví dụ — Hệ thống đặt vé máy bay

### SeatAllocationService — phân bổ ghế

Logic: Tìm ghế phù hợp cho hành khách dựa trên hạng vé, ưu tiên, trạng thái ghế. Logic này không thuộc về Reservation (chưa có ghế) cũng không thuộc về Flight (không biết về preference của hành khách).

```java
public class SeatAllocationService {

    public SeatId allocate(FlightId flightId, SeatClass seatClass,
                           SeatPreference preference, FlightRepository flightRepo) {
        Flight flight = flightRepo.findById(flightId);

        List<Seat> availableSeats = flight.getAvailableSeats(seatClass);
        if (availableSeats.isEmpty()) {
            throw new NoSeatsAvailableException(flightId, seatClass);
        }

        // Business rule: ưu tiên ghế cửa sổ/lối đi theo preference
        return availableSeats.stream()
            .filter(seat -> seat.matches(preference))
            .findFirst()
            .orElse(availableSeats.get(0))  // fallback: ghế đầu tiên còn trống
            .getId();
    }
}
```

### PricingService — tính giá vé

Logic: Tính giá dựa trên hạng vé, số ngày trước chuyến bay, số ghế còn lại. Không thuộc về Flight hay Reservation.

```java
public class PricingService {

    public Money calculatePrice(FlightId flightId, SeatClass seatClass,
                                LocalDate purchaseDate, FlightRepository flightRepo) {
        Flight flight = flightRepo.findById(flightId);
        Money basePrice = flight.getBasePrice(seatClass);

        long daysUntilDeparture = ChronoUnit.DAYS.between(purchaseDate, flight.getDepartureDate());
        int availableSeats = flight.getAvailableSeatCount(seatClass);

        // Business rule: giá tăng khi gần ngày bay và còn ít ghế
        BigDecimal multiplier = BigDecimal.ONE;

        if (daysUntilDeparture <= 3) {
            multiplier = multiplier.multiply(new BigDecimal("1.5"));
        } else if (daysUntilDeparture <= 7) {
            multiplier = multiplier.multiply(new BigDecimal("1.2"));
        }

        if (availableSeats <= 5) {
            multiplier = multiplier.multiply(new BigDecimal("1.3"));
        }

        return basePrice.multiply(multiplier);
    }
}
```

### ReservationTransferPolicy — policy chuyển nhượng đặt chỗ

Logic: Kiểm tra một đặt chỗ có được phép chuyển nhượng cho người khác không. Đây là business policy liên quan đến cả Reservation và Passenger.

```java
public class ReservationTransferPolicy {

    public boolean canTransfer(Reservation reservation, Passenger newPassenger) {
        // Không transfer được nếu đã confirm quá 24h
        if (reservation.isConfirmedMoreThan(Duration.ofHours(24))) {
            return false;
        }
        // Không transfer được nếu đây là vé hạng Economy Promotion
        if (reservation.getFareType() == FareType.ECONOMY_PROMO) {
            return false;
        }
        // Không transfer cho người dùng bị blacklist
        if (newPassenger.isBlacklisted()) {
            return false;
        }
        return true;
    }
}
```

## 4. Domain Service vs Application Service

| | Domain Service | Application Service |
|--|---------------|---------------------|
| **Chứa** | Business logic thuần | Điều phối flow |
| **Biết về** | Entity, Value Object, Repository | Domain Service, Repository, External Services |
| **Biết về HTTP/Queue** | Không | Có |
| **Testable** | Dễ test thuần, không cần mock nhiều | Cần mock nhiều hơn |
| **Ví dụ** | PricingService, SeatAllocationService | BookFlightUseCase, ProcessPaymentUseCase |

```java
// Application Service — điều phối, không chứa business logic
public class BookFlightUseCase {
    private final SeatAllocationService seatAllocationService;  // Domain Service
    private final PricingService pricingService;                // Domain Service
    private final ReservationRepository reservationRepo;

    public ReservationId execute(BookFlightCommand command) {
        // Dùng Domain Service để tính giá và phân bổ ghế
        Money price = pricingService.calculatePrice(
            command.flightId(), command.seatClass(), LocalDate.now(), flightRepo
        );
        SeatId seat = seatAllocationService.allocate(
            command.flightId(), command.seatClass(), command.preference(), flightRepo
        );

        // Tạo Aggregate
        Reservation reservation = new Reservation(command.flightId(), seat, price);
        reservationRepo.save(reservation);

        return reservation.getId();
    }
}
```

## 5. Lưu ý

- Đừng nhầm Domain Service với Service layer trong kiến trúc layered (ví dụ Spring `@Service`)
- Domain Service **không** inject HttpClient, EmailSender, hay bất cứ thứ gì infrastructure
- Nếu Service có quá nhiều logic → xem xét lại xem logic đó có nên về Entity/Value Object không
- Đặt tên rõ ràng theo domain: `PricingService`, `SeatAllocationService`, không phải `FlightService` (quá chung)
