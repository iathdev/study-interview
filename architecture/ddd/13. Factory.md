# Factory

Factory chịu trách nhiệm **tạo ra các Aggregate và Entity phức tạp**. Khi logic khởi tạo một object quá phức tạp để đặt trong constructor, Factory giúp encapsulate quá trình đó.

> Factory giải phóng Aggregate khỏi việc biết cách tạo ra chính mình.

---

## 1. Khi nào cần Factory

- Constructor cần quá nhiều tham số → khó đọc, dễ nhầm
- Tạo Aggregate cần gọi nhiều service/repository
- Logic tạo object phức tạp, có nhiều biến thể
- Muốn đặt tên rõ ràng cho các cách khởi tạo khác nhau

---

## 2. Ví dụ — Hệ thống đặt vé máy bay

### ReservationFactory — tạo đặt chỗ mới

Tạo một Reservation cần: validate flight còn ghế, tính giá, tạo các ReservationItem — quá nhiều logic để nhét vào constructor.

```java
public class ReservationFactory {

    private final FlightRepository flightRepo;
    private final PricingService pricingService;
    private final SeatAllocationService seatAllocationService;

    public Reservation createReservation(CreateReservationCommand command) {
        Flight flight = flightRepo.findById(command.flightId())
            .orElseThrow(() -> new FlightNotFoundException(command.flightId()));

        if (!flight.isAvailable()) {
            throw new FlightNotAvailableException(command.flightId());
        }

        // Tính giá và phân bổ ghế qua Domain Service
        Money price = pricingService.calculatePrice(
            command.flightId(), command.seatClass(), LocalDate.now(), flightRepo
        );
        SeatId seatId = seatAllocationService.allocate(
            command.flightId(), command.seatClass(), command.preference(), flightRepo
        );

        // Tạo Aggregate — constructor gọn hơn vì Factory đã chuẩn bị đủ dữ liệu
        Reservation reservation = new Reservation(
            ReservationId.generate(),
            command.flightId(),
            command.passengerId(),
            seatId,
            price,
            ReservationStatus.HELD,
            LocalDateTime.now().plusMinutes(15)  // hold 15 phút
        );

        return reservation;
    }

    // Named constructor — rõ ràng hơn về use case
    public Reservation createGroupReservation(GroupBookingCommand command) {
        // logic cho đặt vé nhóm (>= 10 người, có giảm giá)
        ...
    }
}
```

### Static Factory Method — trên chính Aggregate

Khi logic đơn giản hơn, có thể dùng static factory method trên Aggregate Root:

```java
public class Flight {

    // Private constructor — bên ngoài không thể new trực tiếp
    private Flight(FlightId id, String number, AirportCode origin,
                   AirportCode destination, LocalDateTime departure) {
        ...
    }

    // Static factory method — tên rõ ràng hơn constructor
    public static Flight schedule(String flightNumber, AirportCode origin,
                                  AirportCode destination, LocalDateTime departure) {
        validateRoute(origin, destination);
        return new Flight(FlightId.generate(), flightNumber, origin, destination, departure);
    }

    public static Flight reconstitute(FlightId id, String number, ...) {
        // Dùng khi restore từ DB — không cần validate vì data đã valid
        return new Flight(id, number, ...);
    }
}

// Dùng
Flight flight = Flight.schedule("VN123", AirportCode.HAN, AirportCode.SGN, departureTime);
```

## 3. Factory vs Constructor

| | Constructor | Factory |
|--|-------------|---------|
| **Khi nào dùng** | Logic tạo đơn giản | Logic tạo phức tạp |
| **Dependency** | Không inject service | Có thể dùng service |
| **Tên** | Tên class | Tên mô tả use case |
| **Biến thể** | Khó đọc nếu nhiều biến thể | Mỗi biến thể có tên riêng |

---

## 4. Factory trong Reconstitution

Khi load từ database, cần tạo lại Aggregate mà **không chạy lại business validation** (vì data đã valid):

```java
public class ReservationMapper {

    // Reconstitute — không validate, chỉ rebuild object từ data
    public Reservation toDomain(ReservationJpaEntity entity) {
        return Reservation.reconstitute(
            new ReservationId(entity.getId()),
            new FlightId(entity.getFlightId()),
            new PassengerId(entity.getPassengerId()),
            new SeatId(entity.getSeatId()),
            new Money(entity.getPrice(), entity.getCurrency()),
            ReservationStatus.valueOf(entity.getStatus()),
            entity.getHeldUntil()
        );
    }
}
```

Tách biệt `create` (validate) và `reconstitute` (rebuild) là best practice quan trọng.

---

## 5. Lưu ý

- Factory nên nằm ở **domain layer** nếu logic là business logic
- Factory nằm ở **application layer** nếu cần inject infrastructure service
- Đừng để Factory biết về persistence (không gọi repository bên trong Factory)
- Factory không lưu object vào DB — đó là việc của Repository
