# Domain Event

Domain Event là **bản ghi bất biến về một điều gì đó đã xảy ra trong domain**. Nó mô tả một sự kiện quan trọng theo ngôn ngữ nghiệp vụ, ở thì quá khứ.

> "Reservation đã được xác nhận" — không phải "hãy gửi email" hay "hãy cộng điểm". Domain Event chỉ nói điều gì đã xảy ra, không ra lệnh ai phải làm gì.

---

## 1. Đặc điểm

| Đặc điểm | Mô tả |
|----------|-------|
| **Immutable** | Không thay đổi sau khi tạo — đã xảy ra rồi, không sửa được |
| **Thì quá khứ** | Tên event dùng quá khứ: `ReservationConfirmed`, không phải `ConfirmReservation` |
| **Có timestamp** | Ghi lại thời điểm xảy ra |
| **Thuộc về Aggregate** | Được raise bởi Aggregate Root khi có state transition |
| **Bất đồng bộ** | Consumer xử lý event độc lập, không blocking |

---

## 2. Cấu trúc

```java
public abstract class DomainEvent {
    private final UUID eventId;
    private final LocalDateTime occurredOn;

    protected DomainEvent() {
        this.eventId = UUID.randomUUID();
        this.occurredOn = LocalDateTime.now();
    }
}

// Event cụ thể — chỉ chứa đủ thông tin cần thiết
public class ReservationConfirmed extends DomainEvent {
    private final ReservationId reservationId;
    private final FlightId flightId;
    private final PassengerId passengerId;
    private final Money totalPrice;

    // Constructor + getters, không có setter
}

public class ReservationCancelled extends DomainEvent {
    private final ReservationId reservationId;
    private final CancellationReason reason;
}

public class FlightDelayed extends DomainEvent {
    private final FlightId flightId;
    private final Duration delay;
    private final LocalDateTime newDepartureTime;
}
```

## 3. Luồng hoạt động

```
Aggregate Root
    │
    │ state transition (confirm, cancel, expire...)
    │
    ▼
registerEvent(new ReservationConfirmed(...))   ← lưu tạm trong memory
    │
    │ Application Service save xong
    │
    ▼
eventPublisher.publish(event)                  ← đẩy ra message bus
    │
    ├──▶ Notification BC → gửi email xác nhận
    ├──▶ Loyalty BC      → cộng điểm
    └──▶ Analytics BC    → ghi nhận booking
```

**Quan trọng:** Aggregate chỉ **đăng ký** event trong memory. Application Service mới **publish** ra ngoài — sau khi save thành công. Điều này đảm bảo event chỉ được gửi khi transaction đã commit.

```java
// Application Service
@Transactional
public void confirmReservation(ConfirmReservationCommand cmd) {
    Reservation reservation = repo.findById(cmd.reservationId());

    reservation.confirm(cmd.payment());  // aggregate đăng ký event nội bộ
    repo.save(reservation);              // commit transaction trước

    reservation.pullEvents()             // lấy event ra và publish
               .forEach(eventPublisher::publish);
}
```

## 4. Domain Event vs Command

| | Command | Domain Event |
|--|---------|-------------|
| **Ý nghĩa** | Yêu cầu làm gì đó | Thông báo điều gì đã xảy ra |
| **Thì** | Mệnh lệnh: `ConfirmReservation` | Quá khứ: `ReservationConfirmed` |
| **Kết quả** | Có thể bị reject | Đã xảy ra, không thể hủy |
| **Chiều** | Gửi đến một handler | Broadcast đến nhiều consumer |
| **Ví dụ** | `CancelReservationCommand` | `ReservationCancelled` |

---

## 5. Domain Event vs Integration Event

| | Domain Event | Integration Event |
|--|-------------|------------------|
| **Phạm vi** | Nội bộ trong một BC | Gửi ra ngoài cho BC khác |
| **Model** | Dùng domain type (`ReservationId`, `Money`) | Dùng primitive, serializable |
| **Versioning** | Không cần strict | Cần backward compatible |
| **Transport** | In-memory hoặc message bus | Message bus (Kafka, RabbitMQ) |

Thông thường Domain Event được **chuyển đổi** thành Integration Event trước khi gửi ra ngoài:

```java
// Domain Event — nội bộ BC
public class ReservationConfirmed extends DomainEvent {
    private final ReservationId reservationId;
    private final Money totalPrice;
}

// Integration Event — gửi ra Kafka cho các BC khác
public class ReservationConfirmedIntegrationEvent {
    private final String reservationId;  // primitive
    private final BigDecimal amount;
    private final String currency;
    private final long occurredOnEpoch;
}
```

## 6. Outbox Pattern

Đảm bảo Domain Event không bị mất khi service crash sau khi commit DB nhưng trước khi publish lên Kafka.

Chi tiết xem tại: [Event-Driven Architecture — Outbox Pattern](../../architecture/event-driven-architecture.md)

## 7. Ví dụ — Tất cả event trong hệ thống đặt vé máy bay

| Event | Raised by | Consumer |
|-------|-----------|----------|
| `ReservationHeld` | Reservation | Notification (gửi reminder), Analytics |
| `ReservationConfirmed` | Reservation | Notification (email xác nhận), Loyalty (cộng điểm), Analytics |
| `ReservationCancelled` | Reservation | Notification (email hủy), Inventory (giải phóng ghế) |
| `ReservationExpired` | Reservation | Inventory (giải phóng ghế), Analytics |
| `FlightDelayed` | Flight | Notification (thông báo hành khách), Reservation (cập nhật) |
| `FlightCancelled` | Flight | Notification, Reservation (auto cancel + hoàn tiền) |
| `PaymentCompleted` | Payment | Reservation (confirm), Notification |
| `PaymentFailed` | Payment | Reservation (cancel), Notification |
