# Repository

Repository là **abstraction cho việc lưu trữ và truy xuất Aggregate**. Nó che giấu hoàn toàn chi tiết persistence (SQL, NoSQL, cache...) khỏi domain layer — domain chỉ cần biết "lưu" và "lấy", không cần biết lưu ở đâu.

> Repository làm cho domain layer coi persistence như một **collection trong bộ nhớ**.

---

## 1. Đặc điểm

| Đặc điểm | Mô tả |
|----------|-------|
| **Một Repository per Aggregate Root** | Không có repository cho Entity con |
| **Interface trong domain** | Implementation nằm ở infrastructure layer |
| **Collection-like API** | `findById`, `save`, `delete` — không lộ SQL |
| **Trả về Aggregate** | Không trả về raw data hay DTO |

---

## 2. Ví dụ — Interface trong Domain Layer

```java
// Domain layer — chỉ có interface, không có implementation
public interface ReservationRepository {
    Optional<Reservation> findById(ReservationId id);
    List<Reservation> findByFlightId(FlightId flightId);
    List<Reservation> findHeldReservationsExpiredBefore(LocalDateTime cutoff);
    void save(Reservation reservation);   // cả insert và update
    void delete(ReservationId id);
}

public interface FlightRepository {
    Optional<Flight> findById(FlightId id);
    List<Flight> findByRoute(AirportCode origin, AirportCode destination, LocalDate date);
    void save(Flight flight);
}

public interface PassengerRepository {
    Optional<Passenger> findById(PassengerId id);
    Optional<Passenger> findByEmail(Email email);
    void save(Passenger passenger);
}
```

## 3. Implementation trong Infrastructure Layer

```java
// Infrastructure layer — implementation cụ thể với JPA
@Repository
public class JpaReservationRepository implements ReservationRepository {

    private final ReservationJpaRepo jpaRepo;  // Spring Data JPA repo
    private final ReservationMapper mapper;

    @Override
    public Optional<Reservation> findById(ReservationId id) {
        return jpaRepo.findById(id.getValue())
            .map(mapper::toDomain);  // JPA Entity → Domain Aggregate
    }

    @Override
    public void save(Reservation reservation) {
        ReservationJpaEntity entity = mapper.toJpa(reservation);  // Domain → JPA Entity
        jpaRepo.save(entity);
    }

    @Override
    public List<Reservation> findHeldReservationsExpiredBefore(LocalDateTime cutoff) {
        return jpaRepo.findByStatusAndHeldUntilBefore(ReservationStatus.HELD, cutoff)
            .stream()
            .map(mapper::toDomain)
            .collect(toList());
    }
}
```

## 4. Repository vs DAO

| | Repository | DAO (Data Access Object) |
|--|-----------|--------------------------|
| **Tư duy** | Collection of domain objects | Table/query abstraction |
| **Trả về** | Domain Aggregate | Raw data, DTO, hoặc ORM entity |
| **Granularity** | Per Aggregate Root | Per table |
| **Phụ thuộc domain** | Có (biết về Aggregate) | Không (chỉ biết về data) |
| **Dùng trong** | DDD | Mọi kiến trúc |

```java
// DAO — expose SQL/table detail ra ngoài
public interface ReservationDao {
    List<Map<String, Object>> findByStatus(String status);  // raw data
    int updateStatus(Long id, String status);               // SQL-level operation
}

// Repository — domain-oriented
public interface ReservationRepository {
    List<Reservation> findPendingReservations();            // domain language
    void save(Reservation reservation);                     // aggregate-level
}
```

## 5. Không có Repository cho Entity con

```java
// Tệ — ReservationItem là Entity con của Reservation
// Không bao giờ được có repository riêng cho nó
public interface ReservationItemRepository {  // ← SAI
    ReservationItem findById(Long id);
}

// Đúng — truy cập qua Aggregate Root
Reservation reservation = reservationRepo.findById(reservationId);
ReservationItem item = reservation.getItem(passengerId);
```

## 6. Query phức tạp — CQRS

Repository trong DDD dùng cho **write side** (command). Với query phức tạp (reports, dashboard), nên tách ra:

```java
// Write side — Repository (domain-oriented)
public interface ReservationRepository {
    Optional<Reservation> findById(ReservationId id);
    void save(Reservation reservation);
}

// Read side — Query Service (không qua domain model, thẳng vào DB)
public interface ReservationQueryService {
    ReservationSummaryDto getReservationSummary(ReservationId id);
    Page<ReservationListDto> searchReservations(ReservationSearchCriteria criteria);
    FlightStatisticsDto getFlightStatistics(FlightId flightId);
}
```

Query Service có thể dùng native SQL, View, hoặc đọc từ read database riêng (Event Sourcing + Projection).

---

## 7. Lưu ý

- **Không để business logic trong Repository**: Repository chỉ tìm kiếm và lưu, không quyết định business rule.
- **Tránh Generic Repository**: `GenericRepository<T>` thường bị lạm dụng, tốt hơn là interface cụ thể cho từng Aggregate.
- **Unit of Work**: Nhiều Aggregate thay đổi trong một request → dùng Unit of Work pattern để đảm bảo transaction.
