## 🎓 1. Trợ lý học tập ảo là gì?

**AI Tutor / Chatbot** là một trợ lý có thể:

* Trả lời câu hỏi học viên về nội dung bài học
* Giải thích lại khái niệm khó (dựa trên kiến thức giảng viên / sách)
* Hỏi ngược học viên để kiểm tra kiến thức
* Theo dõi tiến độ và gợi ý hành động tiếp theo

📌 **Mục tiêu:** Tạo trải nghiệm học “có người kèm”, 24/7, tăng động lực và hiểu sâu hơn.

## 🔍 2. Các bài toán thực tế

| Bài toán cụ thể                | Mô tả                                          | AI xử lý thế nào                                                 |
| ------------------------------ | ---------------------------------------------- | ---------------------------------------------------------------- |
| 💬 Giải thích nội dung bài học | Học viên hỏi: “Tại sao dùng if else ở đây?”    | AI truy nội dung khóa học, giải thích lại bằng ngôn ngữ đơn giản |
| ❓ Trả lời câu hỏi lý thuyết    | Học viên hỏi: “Hệ điều hành là gì?”            | AI truy kiến thức đã được index hoặc gọi LLM như GPT             |

| 🧭 Gợi ý học tiếp              | “Bạn nên làm bài tập số 3 tiếp theo”           | Dựa trên tiến độ, hành vi học, kết quả quiz                      |
| 🗂 Truy cập nội dung           | “Cho mình slide bài 2”                         | AI tìm đúng tài liệu hoặc video trong hệ thống                   |

## ⚙️ 4. Cách giao tiếp kỹ thuật

### ✅ REST API (phổ biến, realtime)

| API                 | Mô tả                                   |
| ------------------- | --------------------------------------- |
| `POST /ask`         | Gửi câu hỏi từ học viên → AI trả lời    |
| `POST /next-action` | Gợi ý học phần tiếp theo                |
| `GET /quiz/suggest` | Sinh câu hỏi kiểm tra kiến thức vừa học |

**Ưu điểm:** Phản hồi nhanh, dễ kiểm soát
**Nhược:** Không thích hợp khi AI cần thời gian lâu để xử lý → dùng Webhook hoặc polling.

---

### ✅ Webhook (khi AI cần thời gian xử lý dài)

1. LMS gọi `POST /ask`, nhận về `task_id`
2. AI xử lý async
3. Khi có kết quả, AI `POST /webhook/answer` về LMS (do LMS cung cấp)

📌 Dùng cho xử lý phức tạp, như: Tóm tắt bài học, hỏi phân tích biểu đồ, sinh video giải thích,...

---

### ✅ Event-driven (ít dùng cho chatbot, nhưng dùng khi cần log)

* Mỗi câu hỏi được gửi đồng thời vào Kafka → AI học từ đó để cải thiện mô hình
* Hoặc AI gửi hành vi học viên vào Kafka cho LMS ghi nhận log

## 🧱 5. Triển khai AI Tutor gồm gì?

| Thành phần        | Công nghệ gợi ý                                             |
| ----------------- | ----------------------------------------------------------- |
| LLM engine        | OpenAI GPT, Claude, Mistral, Llama, v.v.                    |

| NLP pipeline      | LangChain, Haystack, llama-index                            |
| State management  | Redis session (chat), vector memory                         |
| Prompting         | "Bạn là trợ lý học Python, hãy trả lời..."                  |

## 🔐 6. Bảo mật trong AI Tutor

* 🧑‍💻 Xác thực người dùng bằng JWT
* 📖 Giới hạn nội dung truy cập theo quyền user (không cho học viên hỏi khóa chưa mua)
* 🔍 Kiểm tra câu hỏi đầu vào (chống prompt injection, spam, abuse)
* ⏳ Timeout: Chặn việc treo xử lý AI quá lâu
* ✅ Audit log: Ghi lại câu hỏi – trả lời để kiểm tra sau

## 🧪 7. Case Study giả định

### 🧩 Tình huống:

Học viên tên Lan đang học khóa "Lập trình Python cơ bản", vừa học xong bài "Hàm trong Python".

* Lan hỏi: “Tham số mặc định là gì?”
* LMS frontend gọi API `POST /ask`, body:

```json
{
  "userId": 1234,
  "courseId": "python-101",
  "context": "functions",
  "question": "Tham số mặc định là gì?"
}
```

* AI:

  1. Truy lại bài giảng "functions" từ LMS
  2. Gửi context + câu hỏi vào LLM
  3. Trả lại:

     > “Tham số mặc định là giá trị được gán sẵn cho tham số nếu bạn không truyền vào. Ví dụ: `def hello(name='Lan')`.”

* LMS hiển thị câu trả lời
